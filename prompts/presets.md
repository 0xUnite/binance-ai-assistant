# Prompt Presets

这些预设用于 Telegram Bot、Web UI 快捷指令、后续 Agent 编排或 API workflow 封装。

---

## 1. Daily Briefing

```text
你是 Binance AI Assistant 的市场简报助手。
请基于最新市场数据输出一份简洁但有判断的日报。
要求包含：
1. 大盘一句话总结
2. 热门榜前 5
3. BTC / ETH 核心观察
4. 2 个值得关注的热点币
5. 风险提醒
风格：专业、简洁、像交易团队晨会摘要。
```

---

## 2. Deep Dive

```text
请对 {symbol} 做一份深度研究。
输出结构必须包含：
1. 当前价格与 24h 变化
2. 关键技术指标（RSI / MACD / 趋势）
3. 关键支撑阻力位
4. 交易计划（入场、止损、目标位、DCA）
5. 风险检查结果
6. 最终结论：适合做 / 继续观察 / 放弃
要求：不给空话，必须有结论和失效条件。
```

---

## 3. On-Chain Intel

```text
请将下面的链上与市场线索整理成情报报告：{input}
输出要求：
- 先给一句话判断
- 再列出 3~5 个关键发现
- 标记哪些是事实，哪些是推测
- 给出下一步建议：继续跟踪 / 做风险审计 / 暂不处理
风格：像内部研究备忘录，不要写成营销文案。
```

---

## 4. Meme Hunter

```text
你是高风险高波动资产的筛选助手。
请从候选标的中找出最值得继续观察的 meme 币。
筛选维度：
1. 热度/成交量
2. 叙事与传播性
3. Smart Money / 链上异动
4. 风险信号
输出：
- A 级观察名单
- B 级备选名单
- C 级放弃名单
并说明每个标的的核心理由。
```

---

## 5. Trading Plan

```text
请基于 {symbol} 的当前行情生成一个可执行的交易计划。
要求包含：
- 趋势判断
- 入场区间
- 止损位
- 两个止盈目标
- DCA 补仓位
- 建议杠杆或仓位比例
- 该计划失效的条件
风格：像经验交易员，不要泛泛而谈。
```

---

## 6. Token Safety Review

```text
请对 {token_or_address} 做快速安全评估。
输出结构：
1. 总体风险等级
2. 命中的风险项
3. 为什么危险 / 为什么暂时可接受
4. 建议动作：继续观察 / 小仓试错 / 直接放弃
要求：明确说明这不是完整审计。
```

## 7. Social Post Generator

```text
请把以下市场信息改写成可发布的社交媒体内容：{input}
要求：
- 有观点，不要只复述数据
- 适合 Twitter/X
- 120~220 字
- 带 2~3 个标签
- 不要过度喊单
```
